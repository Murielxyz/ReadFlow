### 🔧 核心架构修复：SQL 兼容性

**问题**：expo-sqlite 不支持 `SUM()`、`COUNT()`、`GROUP BY`、`UNION ALL`、`date()`、`strftime()`、`ORDER BY ... LIMIT` 等 SQL 函数。

**解决方案**：全部改为 `SELECT *` → JS `filter`/`reduce`/`Map`/`Set` 计算。

| 文件 | 改动 |
|------|------|
| `useToday.ts` | 今日时长/页数/读过书籍/连续天数/hero 时长 → 全量查询 + JS 计算 |
| `statistics.tsx` | 统计卡片/图表/日历/排行 → 全量 + JS 分组聚合 |
| `my-reading.tsx` | 书籍列表时长 → `SELECT *` + JS Map 累加 |
| `reading-books.tsx` | 阅读书籍页 → 同上 |
| `useSourceStore.ts` | 来源统计 → 同上 |
| `useBookStore.ts` | 评分清洗 `typeof rating !== 'number'` → null |

---

### 🕐 时区统一：localISO()

**问题**：`new Date().toISOString()` 产生 UTC 时间，与中国时区（UTC+8）差 8 小时，导致凌晨记录对不上"今天"。

**解决**：新增 `localISO()` / `localDate()` 工具函数，10 个 Store 全部替换。
