## 一、阅读器功能

### 问题 1：导入本地电子书后无法打开阅读

**实际验证**：
- 阅读器使用 Papyrus SDK（`@papyrus-sdk/engine-native`），**非 WebView+epub.js**
- `ImportModal` 通过 `DocumentPicker` 选文件 → `importFileCache` 模块缓存
- `add-book.tsx` 创建书 + `addSource` 存储 `file_uri`
- `SourceCard` 点击 `开始阅读` → `router.push('/reader/${source.id}')`
- `reader/[bookId].tsx` 用 `bookId`（实际是 sourceId）查询 `reading_sources` → 取 `file_uri` → `engine.load({ source: { uri } })`
- ⚠️ 链路问题：`DocumentPicker` 返回 `content://` URI（Android），Papyrus 可能不支持
- ⚠️ 命名问题：路由参数叫 `bookId` 实际存的是 `sourceId`

**修复方案**：
1. 打印日志确认 `file_uri` 值
2. 如果 `content://` URI 不可用，改用 `expo-file-system` 复制到应用内部存储
3. `ImportModal` 已有 `copyToCacheDirectory: true` 但文件可能仍需复制到可持久化位置

## 二、在线搜索

### 问题 2：搜索书籍后未保存却出现在书架

**实际验证**：✅ **确认存在**
- `add-book.tsx` 第 156-179 行：`isPrefill` 模式下 `useEffect` **自动调用 `addBook()`**
- 用户进入表单 → 已自动创建书籍 → 即使点返回也已在书架

**修复方案**：
1. 删除 `prefillReady` 自动创建逻辑
2. 预填模式仅预填表单字段，**不自动创建**
3. 按钮文字「保存修改」→「添加书籍」
4. 点击「添加书籍」才调用 `addBook()` + Alert 提示

## 四、书籍颜色统一

### 问题 5：封面颜色不统一

**实际验证**：✅ **确认存在**
- `useBookStore.addBook` 第 48 行：`colorIndex = hash(title) % BOOK_ACCENT_COLORS.length` → 根据书名 hash 随机选色
- 不同页面不同时间创建的书颜色会不同

**修复方案**：`accent_color` 统一固定为紫色 `#7C6BFF`

## 五、导入元数据

### 问题 6：导入 EPUB 没有识别封面和作者

**实际验证**：✅ **确认存在问题**
- `extractMetadata.ts` 存在且有提取逻辑
- ⚠️ `extractEpubMetadata` 使用 `FileSystem.readAsStringAsync`（`expo-file-system/legacy`）
- ⚠️ EPUB 是 ZIP 压缩包，直接读文本无法解析 ZIP，需解压后读取 `content.opf`
- ⚠️ `ImportModal` 用 `DocumentPicker` 的 `copyToCacheDirectory: true`，文件在临时目录

**修复方案**：
1. 改用更可靠的 EPUB 解析方式（或从文件名提取信息作为降级）
2. 当前文件名提取已工作（`extractTitleFromFilename`）
